文章系统使用指南

本站的文章系统基于 MDX 文件自动化构建。在网站的构建阶段会自动扫描并解析位于 src/pages/posts/_posts 目录下的所有 MarkdownX (.mdx) 格式文件，并将其转换为对应的网页页面。

MDX 是 Markdown 语法的扩展，集成了 JSX 功能。我们这里主要使用的是 mdx 的 frontmatter 特性来管理文章的元数据。frontmatter 位于文件的最顶部，是由两行---包裹的 YAML 格式数据区域。

frontmatter 定义的 Metadata 结构如下：

PropStatusTypeExampleDescribe
titleRequiredStringtitle: Example Post文章标题
descriptionRequiredStringtitle: Description of the post.文章简介
pubDateRequiredStringpubDate: 2025-04-29发布日期。格式为YYYY-MM-DD
modDateOptionalStringmodDate: 2025-04-29最后修改日期

请务必正确填写所有标记为 Required 的必填字段，这是文章系统正常工作的必要条件。

以下是一个包含可选字段的完整 frontmatter 示例。只需将这部分内容添加到现有的 .md 文件最顶端即可：

---
title: Example Post
description: The description of this article.
pubDate: 2025-04-29
modDate: 2025-05-01
---

然后请将文件的扩展名从 .md 更改为 .mdx。

  • 文件命名规范：

文件命名至关重要，因为它将直接映射为文章网页的 URL 路径。请使用简洁且具有描述意义的全小写英文字词(以及数字)，并用连字符(-)连接单词(而不是使用下划线_)，例如example-post.mdx.

有关更详细的 URL 结构推荐阅读 URL structure best practices for Google.
